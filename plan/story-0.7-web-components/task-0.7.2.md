# Task 0.7.2: Component Variants

## Description
Implement different display variants for the widget: toggle and checkbox. Each variant should have appropriate UI and interaction patterns based on configuration.

## Objectives
- Implement toggle variant UI
- Implement checkbox variant UI
- Switch rendering based on variant property
- Style both variants
- Ensure accessibility for both

## Acceptance Criteria
- ✅ Toggle variant displays switch UI
- ✅ Checkbox variant displays checkbox UI
- ✅ Variant switches based on property
- ✅ Both variants show quote price
- ✅ Both variants show protection copy
- ✅ Styles appropriate for each variant
- ✅ Both variants keyboard accessible
- ✅ 90%+ test coverage

## Variant Content
**Toggle:**
- Switch control (on/off)
- Protection copy
- Price display
- Coverage details

**Checkbox:**
- Checkbox control
- Protection copy
- Price display
- Coverage details

## Testing Strategy
- Test toggle variant renders
- Test checkbox variant renders
- Test variant switching
- Test price display in both
- Test keyboard interaction

## Dependencies
- Base component (task 0.7.1)
